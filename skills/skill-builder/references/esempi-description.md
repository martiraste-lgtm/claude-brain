# Esempi di Description Efficaci

## Struttura

```
[COSA fa] + [QUANDO usarla] + [Trigger phrases] + [Esclusioni opzionali]
```

**Lunghezza ideale:** 150-300 caratteri | **Massimo:** 1024 caratteri

---

## Categoria 1: Document & Asset Creation

```yaml
# ✅ OTTIMA
description: Genera sales deck professionali con struttura persuasiva.
  Usa quando l'utente dice "crea un pitch deck", "prepara una
  presentazione di vendita", "sales deck per [cliente]".

# ❌ TROPPO VAGA
description: Aiuta con le presentazioni.
```

## Categoria 2: Workflow Automation

```yaml
# ✅ OTTIMA
description: Applica il framework [Nome] per analisi strategica.
  Usa quando l'utente dice "analisi strategica", "applica framework",
  "valutazione [topic]".

# ❌ MANCANO TRIGGER
description: Esegue analisi strategiche approfondite.
```

## Categoria 3: MCP Enhancement

```yaml
# ✅ OTTIMA
description: Gestisce workspace Notion con best practices. Usa quando
  l'utente dice "setup Notion", "crea database Notion", "organizza workspace".
  Requires Notion MCP connection.
```

---

## Pattern per Trigger Phrases

**Verbi:** "crea...", "genera...", "prepara...", "analizza..."
**Contesto:** "aiutami a...", "devo...", "voglio..."
**Oggetti:** "[deliverable] per [contesto]"
