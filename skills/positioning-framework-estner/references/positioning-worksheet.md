# Positioning Worksheet — Da compilare end-to-end

Questa è la worksheet strutturata per costruire il positioning da zero. Compilala in ordine. Non saltare sezioni.

Ogni sezione è uno step. Ogni step alimenta il successivo.

---

## Sezione 1 — Understand the Market

### 👥 Target ICP

Chi è il cliente ideale per te? Chi sta affrontando il challenge che il tuo prodotto risolve?

Struttura minima:
- **Tipo di azienda** (industry, settore, modello di business)
- **Stage / dimensione** (ARR, employees, funding, fase)
- **Regione / geografia** (se rilevante)
- **Persona / ruolo** (chi compra, chi usa, chi influenza)

> _[compila qui]_

---

### 🔑 Use Case (Job-to-be-done)

Cosa vuole fare / ottenere il tuo ICP?

**Formulazione consigliata**: "When [situation], [ICP] wants to [motivation], so they can [expected outcome]."

> _[compila qui]_

---

### 🔥 Core Problem

Perché questo JTBD è difficile oggi? Quale pain strategico o operativo esiste, indipendentemente dalla soluzione?

**Distingui tra**:
- **Pain strategico** (impatta business outcome, crescita, revenue, efficienza)
- **Pain operativo** (impatta workflow quotidiano, tempo, cognitive load)

> _[compila qui]_

---

### 👣 Alternative Solutions — Cluster

Quali tipi di alternative esistono oggi? Crea cluster (non nomi di singoli prodotti, gruppi di approcci).

Struttura:
1. **Cluster 1**: ________________________
   (es. "Software di categoria X", "Tool generalisti", "Agenzie esterne", "Excel/manuale")
2. **Cluster 2**: ________________________
3. **Cluster 3**: ________________________

> _[compila qui]_

---

### 🧱 Dominant Current Solution (Status Quo)

Quale tra i cluster sopra è **dominante** oggi? Come il buyer prova a risolvere il problema attualmente?

Questo diventa il tuo **vero competitor**, anche se non è un software.

> _[compila qui]_

---

### ❌ 3 Key Limitations of the Status Quo

Perché il loro approccio attuale non risolve il problema bene? Scegli le 3 limitazioni più pesanti.

Non scrivere limitazioni generiche ("è lento", "è complesso"). Sii specifico.

1. **Limitation 1**: ________________________
2. **Limitation 2**: ________________________
3. **Limitation 3**: ________________________

**Test di specificità**: ogni limitation risponde a "perché il buyer dice NO allo status quo"? Se la risposta è vaga, rifai.

### ⭐ Competitive Differentiator (Why You)

La differenza strutturale. Cosa ti rende specificamente diverso dalla soluzione dominante?

**Template**:
```
Unlike [dominant current solution], we [competitive differentiator].
```

Esempio:
```
Unlike HubSpot, we offer fast setup, simplicity, and startup-friendly pricing.
```

## Sezione 3 — Articolazione

### 📜 Strategic Positioning Statement (Internal Version)

Questa è la narrativa interna che giustifica perché esisti. 3-5 frasi. Non è marketing copy.

Usa il template specifico per il tuo Primary Anchor (vedi `statement-templates.md`).

> _[compila qui]_

---

### 📣 One-Sentence Positioning Statement

La versione market-facing compressa. Una frase.

Usa il template corrispondente al Primary Anchor.

**Template Activity**:
```
We help [ICP] [manual activity they currently do] so they can [outcome].
```

**Template Use Case**:
```
We help [ICP] [clear job-to-be-done] without [limitations of the status quo].
```

**Template Product Category**:
```
We are a [product category] for [specific ICP].
```

**Template Competitive Alternative**:
```
We are a [product category] alternative to [leader] built for [ICP] with [unique differentiator].
```

## Sezione 5 — Health Check (Self-Assessment)

Passa il positioning finale attraverso queste domande. Se rispondi NO anche solo a una, torna allo step corrispondente.

(Vedi `mistakes-and-health-check.md`.)

- [ ] **Un buyer capisce immediatamente cosa sei?** Se no → Primary Anchor poco chiaro, rivedi `primary-anchor.md`
- [ ] **È ovvio cosa rimpiazzi?** Se no → definisci meglio competitive alternatives, rivedi la sezione Status Quo
- [ ] **La differenziazione è significativa?** Un competitor avrebbe difficoltà a copiarla? Se no → Secondary Angle generico, rivedi `secondary-angles.md`
- [ ] **Evito i 6 common mistakes?** Se no → rivedi `mistakes-and-health-check.md`

---

## Sezione 6 — Validazione esterna

Metti alla prova il positioning con 3 buyer reali. Non amici, non investor, non team. **Buyer veri**.

Domande da fargli:
1. "Cosa fa questo prodotto?" → misura clarity
2. "È per te? Perché sì / no?" → misura fit
3. "Quali alternative vedresti?" → misura coerenza competitiva
4. "Cosa ti convince? Cosa ti lascia dubbioso?" → misura resistenza

**Se almeno 2 su 3 non capiscono cosa fai in 30 secondi**: il positioning non è pronto. Torna alla worksheet.
