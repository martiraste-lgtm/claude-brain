# Libreria Tipi di Slide

10 tipi di slide per caroselli LinkedIn. Ogni tipo ha layout, regole e varianti. Tutti i layout sono per canvas 1080x1350px.

---

## 1. Cover

**Scopo**: prima slide visibile nel feed. Deve catturare attenzione e comunicare il topic in 2 secondi.

**Layout**:
```
+----------------------------------+
|                                  |
|     [TITOLO GRANDE BOLD]         |
|     con keyword evidenziate      |
|                                  |
|  +-------+  +-------+  +------+ |
|  |preview|  |preview|  |previe| |
|  |slide 3|  |slide 7|  |slide | |
|  +-------+  +-------+  +------+ |
|                                  |
|            @handle               |
+----------------------------------+
```

**Regole**:
- Titolo: max 8 parole, bold, 60-80px
- 2-3 keyword evidenziate con colori accento
- Opzionale: collage/preview di 2-3 slide interne (miniature)
- Handle/nome autore in basso
- Nessun body text — solo titolo

**Varianti**:
- **A — Solo titolo**: titolo centrato, grande, nient'altro
- **B — Titolo + collage**: titolo sopra + miniature slide sotto
- **C — Titolo + sottotitolo**: titolo + 1 riga di contesto (max 10 parole)

**Colori tipici**: arancione o coral per highlight del topic principale

## 2. Contesto

**Scopo**: stabilire il problema, la situazione attuale, o il "perche" del carosello. Crea tensione narrativa.

**Layout**:
```
+----------------------------------+
|                                  |
|                                  |
|    [STATEMENT GRANDE]            |
|    su 2-4 righe                  |
|    con 1-2 keyword               |
|    evidenziate                   |
|                                  |
|                                  |
+----------------------------------+
```

**Regole**:
- Statement: 2-4 righe, 36-48px, centrato
- 1-2 keyword evidenziate
- Nessun elemento visivo aggiuntivo — solo testo su bianco
- Tono: provocatorio o sorprendente ("La maggior parte delle aziende fa X. Non funziona.")

**Varianti**:
- **A — Statement puro**: solo testo centrato
- **B — Statement + dato**: statement con 1 numero in evidenza (grande, bold)

**Colori tipici**: coral per problemi, arancione per emphasis

---

## 3. Ponte (Bridge/Transizione)

**Scopo**: pausa di respiro tra sezioni. Riorientare il lettore. Creare anticipazione per la sezione successiva.

**Layout**:
```
+----------------------------------+
|                                  |
|                                  |
|                                  |
|      [FRASE BREVE]              |
|      1-2 righe                   |
|                                  |
|                                  |
|                                  |
+----------------------------------+
```

**Regole**:
- Frase: max 15 parole, 48-60px, bold, centrata
- 1 highlight o 1 sottolineatura (non entrambi)
- Massima aria bianca — la slide deve "respirare"
- Funzione: transizione, domanda retorica, o anticipazione

**Varianti**:
- **A — Domanda**: "Ma come si fa in pratica?"
- **B — Anticipazione**: "Ecco il framework."
- **C — Contrasto**: "Il problema e che..."

**Colori tipici**: crema/giallo per transizioni, arancione per anticipazioni

---

## 4. Confronto

**Scopo**: mostrare A vs B, vecchio vs nuovo, sbagliato vs giusto. Rende visibili le differenze.

**Layout**:
```
+----------------------------------+
|         [TITOLO]                 |
|                                  |
|  +------------+ +------------+  |
|  | COLONNA A  | | COLONNA B  |  |
|  |............| |............|  |
|  | - punto 1  | | - punto 1  |  |
|  | - punto 2  | | - punto 2  |  |
|  | - punto 3  | | - punto 3  |  |
|  +------------+ +------------+  |
|                                  |
+----------------------------------+
```

**Regole**:
- Titolo sezione: max 6 parole, bold
- 2 colonne con card a bordo tratteggiato
- Header colonna: color-coded (coral = negativo/vecchio, verde = positivo/nuovo)
- 3-5 punti per colonna, max 8 parole ciascuno
- Allineamento simmetrico

**Varianti**:
- **A — 2 colonne affiancate**: layout classico left-right
- **B — 2 card impilate**: card A sopra, card B sotto (per confronti piu lunghi)
- **C — Tabella**: griglia con header e righe

**Colori tipici**: coral per colonna negativa, verde per colonna positiva

## 5. Dato

**Scopo**: presentare un dato, una metrica, un numero che sorprende o convince. Autorita attraverso i numeri.

**Layout**:
```
+----------------------------------+
|                                  |
|                                  |
|          [73%]                   |
|     grande, bold, centrato       |
|                                  |
|   delle aziende non ha un        |
|   framework di positioning       |
|                                  |
|                                  |
+----------------------------------+
```

**Regole**:
- Numero: 80-120px, bold, centrato — e l'elemento dominante
- Contesto: 1-2 righe sotto il numero, 28-36px, regular
- Opzionale: piccolo chart/grafico a barre affiancato
- 1 keyword evidenziata nel contesto

**Varianti**:
- **A — Numero singolo**: un dato grande al centro
- **B — Numero + chart**: dato a sinistra, mini grafico a barre/torta a destra
- **C — 2-3 numeri**: griglia con 2-3 dati (ciascuno con label)

**Colori tipici**: arancione per il numero, crema per il contesto

## 6. Punto (Statement + Icona)

**Scopo**: comunicare un singolo concetto con forza visiva. Usata in sequenza per costruire un argomento punto per punto.

**Layout**:
```
+----------------------------------+
|                                  |
|         [ICONA]                  |
|                                  |
|   [STATEMENT con highlight]      |
|   su 2-3 righe                   |
|   e sottolineatura               |
|                                  |
|                                  |
+----------------------------------+
```

**Regole**:
- Icona/stick figure: 60-80px, centrata, sopra il testo
- Statement: 2-3 righe, 36-48px
- 1 highlight + 1 sottolineatura ammessi
- Usare in serie di 2-4 slide consecutive per costruire un argomento

**Varianti**:
- **A — Icona + testo**: icona sopra, statement sotto
- **B — Numero + testo**: numero grande (1, 2, 3...) al posto dell'icona
- **C — Solo statement**: senza icona, piu grande (48-60px)

**Colori tipici**: variabili in base al contenuto semantico

## 7. Dettaglio

**Scopo**: approfondire un concetto con 3 sotto-punti strutturati. Ideale per framework, step, o categorie.

**Layout**:
```
+----------------------------------+
|      [TITOLO SEZIONE]            |
|                                  |
|  +----------------------------+  |
|  | 1  [Label]                 |  |
|  |    Descrizione breve       |  |
|  +----------------------------+  |
|  +----------------------------+  |
|  | 2  [Label]                 |  |
|  |    Descrizione breve       |  |
|  +----------------------------+  |
|  +----------------------------+  |
|  | 3  [Label]                 |  |
|  |    Descrizione breve       |  |
|  +----------------------------+  |
+----------------------------------+
```

**Regole**:
- Titolo: max 6 parole, bold, 40-48px
- 3 card numerate (mai 2, mai 4+ — sempre 3)
- Ogni card: numero con cerchio colorato + label bold + 1-2 righe descrizione
- Card con bordo tratteggiato
- Max 15 parole per card

**Varianti**:
- **A — 3 card verticali**: impilate (layout standard)
- **B — 3 card orizzontali**: affiancate (solo con label molto brevi)
- **C — 3 card con icona**: icona al posto del numero

**Colori tipici**: crema per framework, rosa per categorie, verde per step

## 8. Diagramma

**Scopo**: visualizzare relazioni, strutture, o processi. Rende tangibile cio che e astratto.

**Layout**: variabile in base al tipo di diagramma.

**Sotto-tipi**:

### 8A — Matrice 2x2
```
+----------------------------------+
|      [TITOLO]                    |
|           asse Y                 |
|      +----------+----------+    |
|      |          |          |    |
|      |  quad 1  |  quad 2  |    |
|      |          |          |    |
|      +----------+----------+    |
|      |          |          | asse X
|      |  quad 3  |  quad 4  |    |
|      |          |          |    |
|      +----------+----------+    |
+----------------------------------+
```

### 8B — Diagramma di Venn
```
+----------------------------------+
|         [TITOLO]                 |
|                                  |
|       (  A  )                    |
|     (       (  B  )              |
|       (   )                      |
|                                  |
|   label A    label B             |
|        label intersezione        |
+----------------------------------+
```

### 8C — Flow/Processo
```
+----------------------------------+
|         [TITOLO]                 |
|                                  |
|   [Step 1] --> [Step 2] -->      |
|                                  |
|   --> [Step 3] --> [Step 4]      |
|                                  |
+----------------------------------+
```

### 8D — Nested boxes
```
+----------------------------------+
|  +----------------------------+  |
|  |  +----------------------+  |  |
|  |  |  +----------------+  |  |  |
|  |  |  |   CORE         |  |  |  |
|  |  |  +----------------+  |  |  |
|  |  |      LAYER 2         |  |  |
|  |  +----------------------+  |  |
|  |         LAYER 3            |  |
|  +----------------------------+  |
+----------------------------------+
```

**Regole**:
- Solo label brevi dentro i box (max 3-4 parole)
- Colori accento per differenziare zone
- Frecce per connessioni (nere, 2-3px)
- Titolo: max 6 parole, bold
- Il diagramma e l'elemento dominante — occupa 60-70% della slide

**Colori tipici**: mix di tutti i colori accento per differenziare zone

## 9. Sommario

**Scopo**: consolidare i punti chiave alla fine della sezione. Serve come "cheat sheet" da salvare.

**Layout**:
```
+----------------------------------+
|      [TITOLO: Recap / In        |
|       sintesi / Checklist]       |
|                                  |
|  [Categoria A]                   |
|    - punto 1                     |
|    - punto 2                     |
|                                  |
|  [Categoria B]                   |
|    - punto 3                     |
|    - punto 4                     |
|                                  |
|  [Categoria C]                   |
|    - punto 5                     |
|    - punto 6                     |
+----------------------------------+
```

**Regole**:
- Titolo: "In sintesi", "Recap", "Checklist", o simile
- 2-3 categorie con 2-4 item ciascuna
- Header categoria: evidenziato con colore accento
- Item: brevi (max 8 parole), con checkbox o bullet
- Questa slide viene spesso salvata/screenshottata — deve funzionare standalone

**Varianti**:
- **A — Checklist**: checkbox + item
- **B — Categorizzato**: header + bullet per categoria
- **C — Numerato**: lista ordinata con numeri evidenziati

**Colori tipici**: crema per header categoria, verde per checklist positive

## 10. CTA (Call to Action)

**Scopo**: chiudere il carosello con un'azione specifica. Convertire l'attenzione in follow, subscribe, click.

**Layout**:
```
+----------------------------------+
|                                  |
|                                  |
|   Grazie per aver letto          |
|                                  |
|   [CTA SPECIFICA]                |
|   con highlight                  |
|                                  |
|   @handle                        |
|                                  |
+----------------------------------+
```

**Regole**:
- "Grazie per aver letto" o equivalente: 36px, regular
- CTA specifica: 40-48px, bold, con highlight
- La CTA cambia ogni volta — NON e fissa. Esempi:
  - "Iscriviti alla newsletter 'da 0 al PMF'"
  - "Seguimi per altri framework"
  - "Salva questo carosello per dopo"
  - "Commenta con il tuo caso"
- Handle autore in basso
- Opzionale: link o QR code

**Varianti**:
- **A — Testo semplice**: ringraziamento + CTA
- **B — CTA + box**: CTA dentro un box con bordo tratteggiato
- **C — Multi-CTA**: 2 CTA (primaria bold + secondaria regular)

**Colori tipici**: arancione o verde per highlight CTA
