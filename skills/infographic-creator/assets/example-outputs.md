# Esempi di Output Attesi

Riferimenti rapidi per capire che tipo di output produrre in base al tipo di richiesta.

---

## Esempio A: Processo Sequenziale

**Input utente:** "Ho un processo di onboarding clienti in 5 step: discovery call, proposta, setup, training, go-live"

**Layout scelto:** Timeline Numerata (Layout 5) o Zigzag (Layout 16)

**Estrazione:**
```
Titolo: "5 Step per Onboardare un Nuovo Cliente"
Sottotitolo: "Dal primo contatto al go-live in 30 giorni"

Step 1: Discovery Call
  → Comprendi esigenze, budget, timeline

Step 2: Proposta
  → Presenta soluzione personalizzata, pricing

Step 3: Setup Tecnico
  → Configura account, integrazioni, dati

Step 4: Training Team
  → Forma il team sull'utilizzo, best practices

Step 5: Go-Live
  → Lancio, monitoraggio, supporto dedicato
```

## Esempio B: Framework Gerarchico

```
  AI Tools, Nuovi canali, Automazioni, Test & Learn

Zona centro (arancio) - "Attività Core":

  Community, Newsletter, LinkedIn

Zona base (giallo) - "Fondamenta":
  ICP definito, Positioning chiaro, Messaging consistente,
  Value Proposition, Brand Narrative, Ricerca di mercato
```

## Esempio C: Confronto Modelli

**Input utente:** "Ho 4 modelli di pricing SaaS: freemium, flat rate, usage-based, hybrid. Ognuno ha pro, contro e esempi di aziende"

**Layout scelto:** Tabella con Icone (Layout 14)

**Estrazione:**
```
Titolo: "4 Modelli di Pricing SaaS a Confronto"

| Modello | Come Funziona | Pro | Contro | Esempi |
|---------|--------------|-----|--------|--------|
| 🆓 Freemium | Gratis + premium | Adozione rapida | Conversione bassa | Slack, Notion |
| 💰 Flat Rate | Prezzo fisso/mese | Semplicità | No upside | Basecamp |
| 📊 Usage-Based | Paghi per uso | Scala con valore | Revenue volatile | AWS, Twilio |
| 🔄 Hybrid | Fisso + variabile | Flessibile | Complessità | HubSpot, Snowflake |
```

## Esempio D: Lista Errori

**Input utente:** "Voglio fare un'infografica sui 5 errori più comuni nel content marketing B2B"

**Layout scelto:** Lista Errori a Righe (Layout 12)

**Estrazione:**
```
Titolo: "5 Errori di Content Marketing che Uccidono la Pipeline"

1. Scrivere per tutti → ✕ Messaging generico ✕ Nessuno si identifica
2. Solo contenuti TOFU → ✕ Traffico senza conversioni ✕ Nessun intent
3. Nessun sistema di repurpose → ✕ 1 contenuto = 1 uso ✕ ROI basso
4. Ignorare la distribuzione → ✕ "Pubblica e spera" ✕ Nessuna amplificazione
5. No allineamento con sales → ✕ Lead non qualificati ✕ Sales ignora il marketing
```

## Esempio E: Idea Grezza → Framework

**Input utente:** "Il mio concetto è che per fare GTM devi prima avere le fondamenta (positioning, ICP), poi la macchina (content + outbound), e infine scalare (team, processi, tool)"

**Layout scelto:** Card Numerate a Sezioni (Layout 6)

**Estrazione:**
```
Titolo: "GTM Engine: Da Zero a Scala"

SEZIONE 1 - FONDAMENTA
  1. ICP Research → Definisci chi serve veramente
  2. Positioning → Differenziati nel mercato
  3. Messaging → Parla la lingua del cliente
  4. Value Prop → Articola il valore unico

SEZIONE 2 - MACCHINA
  5. Content System → Crea asset che lavorano per te
  6. Outbound Engine → Raggiungi i prospect giusti
  7. Sales Process → Converti con metodo
  8. Feedback Loop → Impara e adatta

SEZIONE 3 - SCALA
  9. Team Hiring → Le persone giuste nei ruoli giusti
  10. Process Design → Ripetibile e documentato
  11. Tool Stack → Automatizza dove possibile
  12. Metrics → Misura ciò che conta
```
